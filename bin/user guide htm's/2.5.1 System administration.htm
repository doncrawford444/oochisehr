2.5.1 System administration

'Set up' ('System' → 'Set up'):
This feature is only available to the system administrators and users authorized with administrator privileges. 'Set up' manages network URIs and the system administrators' user names and passwords.

'Login attempts allowed' ('System' → 'Set up' → 'Login attempts allowed: '):
There is a maximum number of 'Login attempts allowed' before the system administrator is required to login. The number of login attempts allowed is set by the system administrators in 'System' → 'Set-up'. The 'Login' screen will indicate if the system administrators is required to login, and there is no limit to the number of login attempts for the system administrators. If the system administrator's user name and password does not work (fails to authenticate) for any reason, there is an emergency access method (see "emergency access"). Like the system administrators logins, there is no limit to the number of log in attempts using the emergency access method.

Also, the 'Login' screen's user name and password fields will both automatically clear if two minutes have elapsed without input. The clearing of fields does not count for a login attempt.

'System administrator's user name: ' ('System' → 'Set up' → 'System administrators' user name: '):
Change the system administrator's user name by entering the new name in the field. Each device in a Oochis EHR system must have the same system administrators user name and password. The same system administrators manages multiple devices, as well as user accounts.

'System administrator's password: ' ('System' → 'Set up' → 'System administrators' password: '):
Change the system administrators password by entering the new password in the field. Each device in a Oochis EHR system must have the same system administrators user name and password. The same system administrators manages multiple devices, as well as user accounts.

'System administrator's password (re-enter): ' ('System' → 'Set up' → 'System administrators' password (re-enter): '):
Re-enter the system administrator's new password.

'Program directory URI: ' ('System' → 'Set up' → 'Program directory URI: '):
Enter the directory for program files. Program files are: vocabulary databases, EHR data, provider data (root admin, EHR data, user files, notepads....

enc/: Contains patient EHR data files. Each file contains all the available health information on one and only one patient both recorded and imported. "Available health information," does not necessarily mean all health information. The responsibility of recording and storing EHR data is up to the individual health care providers, and may not be locatable; or, may still be left partially undigitized or unencoded in providers' paper-based charts.

images/: Contains images found throughout the program

lib/: Contains imports (third-party classes not the core java.awt, or java.swing, util...Sun widgets)

medical_code_sets/: Contains encrypted vocabulary meta-thesauri flat database files for: HL-7, LOINC, RXNORM, ICD; and CPT. Also contains a biomoiety thesaurus for 'Decision Support' 'Translational Medicine' functionality.

narrative/: Contains the note pad and content and files imported into the workspace.

sys/: Contains the system administrators' root administrative settings and info.

test_files/: Contains developmental test files.

usr/: Contains user account files.

'Program directory URI: ' ('System' → 'Set up' → 'Network node directory URI: '):
Enter the URI of the network directory (a system directory accessible by all networked devices). This network directory is where system files are located to enable networked devices.

'Claims file URI: ' ('System' → 'Set up' → 'Claims file URI: '):
Enter the URI of where to place a claims folder (a tab-deliminated, computer-readable, text file of medically encoded health data entries). The file is the same as the printable report found in: 'Practice Assistant', 'Reports', 'Coded entries'.

'Browser executable URI: ' ('System' → 'Set up' → 'Browser executable URI: '):
Enter the location of a secured Mozilla Firefox executable. The browser that Oochis EHR uses is a special security-enhanced Internet browser. {{{ships with or without Oochis EHR}}}

'Create desktop icon: ' ('System' → 'Set up' → 'Create desktop icon: '):
A Oochis EHR icon will be placed on the desktop. Alternatively, the icon will be placed in the device's '/user/home/' directory if no '/desktop/' directory is found.
