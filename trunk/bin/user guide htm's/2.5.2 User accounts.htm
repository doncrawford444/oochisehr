2.5.2 User accounts

'User accounts' ('System' → 'User accounts'):
There are three types of user accounts:

Account type: Purpose:
System administrators: manage system settings and user accounts only
Users with administrator privileges: manage health information and user accounts only
Users without administrator privileges: manage health information only

System administrators:
The role of system administrators is to manage system settings and user accounts. System administrators may also be responsible for installing software and maintaining the physical system.

'System' → 'Set-up' (the button as well as the panel) is only available to the system administrators. System administrators manage all accounts in the same networked system. The role of system administrators is to manage every installed device and user account in the system.

Administrator privileges:
  • Users may be authorized with and without administrator privileges in 'System' → 'User accounts' → 'Authorizations'.
  • Users authorized with administrator privileges can be assigned a set of user accounts to manage.
  • Users authorized with administrator privileges can add new user accounts, and authorize them for administrator privileges.
  • Users authorized with administrator privileges can only manage users they create or are assigned by the administrator).
  • System administrators manage all users in the network.

General procedure to assigning user names and passwords to all clinicians:
A system administrator first creates an initial set of users with and without administrator privileges. Then, new users with administrator privileges create further users with and without administrator privileges. This process of creating users with administrator privileges that, in turn, create further users with and without administrator privileges continues until all users (clinicians) are accounted for. 'Authorizations' (see 'Authorizations') are managed only by users with administrator privileges. Users with administrator privileges manage all user accounts they create, and all accounts created by accounts they create.

The roles of system administrators are determined exclusively by the practice. Audit tracking is provided for with logs of system administrator activities in 'Reports' → 'Logs' → 'System-wide activities'.

The information attributed to the system administrators is changeable (such as when a different person assumes the role of system administrators), but the last system administrators account cannot be deleted. The last system administrator cannot be deleted because at least one system administrator is required to add, modify, and delete user accounts. An initial system administrator login user name and password are in the README.TXT file that comes with Oochis EHR.

As per NCO EHR certification criteria, "emergency access," is provided via a special user name and password listed in the README.TXT file packaged with each Oochis EHR release.

Users with administrator privileges: Users with administrator privileges manage user accounts as well as health information. Users with administrator privileges authorize their users to view, create, modify, and delete health information (see Authorizations panel figure).

No 'User accounts' panel for users without administrator privileges: Only system administrators and users with administrator privileges have a 'User accounts' panel in the 'System' window.

Users without administrator privileges: Users without administrator privileges manage health information only. The 'User accounts' button is replaced with the 'User index' button. And users without administrator privileges do not have the 'User accounts' buttons: 'Add user', 'Modify user', and 'Authorizations'.

'User index' ('System' → 'User accounts' → 'User index'):
User index contains the user index and options for a listed index, and a toggles showing deleted users.

'Print' ('System' → 'User accounts' → 'User index' → 'Print'):
Opens a print dialog box for printing.

'Show deleted users' ('System' → 'User accounts' → 'User index' → 'Show deleted users'):
Shows both current and deleted users.

'Full user index' ('System' → 'User accounts' → 'User index' → 'Full user index' ):
Displays the full user index.

'Listed user index' ('System' → 'User accounts' → 'User index' → 'Listed index' ):
Displays the listed user index. The listed user index displays the highlighted users from the full user index. To add users, highlight the users in the listed index. To remove users, highlight the users in the listed index.

'New' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'New'):

'Open' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Open'):
Open a user group.

'Close' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Close'):
Closes the 'Listed index'. If the 'Listed index' is not saved, a prompt will be displayed requesting whether to save any changes before closing.

'Save' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Save'):
Clicking 'Save' saves the 'Listed index'.

'Save as' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Save as'):
Saves the new user group

'Send' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Send' ):
Opens the Messaging window and attaches the user index to a new email.

'Print' ('System' → 'User accounts' → 'User index' → 'Listed index' → 'Print'):
Opens a print dialog box for printing.

'Add user' ('System' → 'User accounts' → 'Add user'):
The user's first name, last name, and title are required to add a user account. Click 'Save' to save the added user.

'Clear' ('System' → 'User accounts' → 'Add user' → 'Clear'):
Clears all fields in the 'Add user' panel.

'Save' ('System' → 'User accounts' → 'Add user' → 'Save'):
Saves and adds the new user to the user index. When saved, the new user is immediately able to login and work on any device in the network. Authorizations may be set for the new user after.

'Authorizations' ('System' → 'User accounts' → 'Authorizations'):
To authorize users with administrator privileges and health information access, first enter the account's user name, then press the keyboard's 'Enter' key or click the 'OK' button.

'All charts' ('System' → 'User accounts' → 'Authorizations' → 'All charts '):
Clicking the 'All charts' button applies the authorizations to all charts.

'Specified charts only' ('System' → 'User accounts' → 'Authorizations' → 'Specified charts only '): Clicking the 'Specified charts only' button applies the authorizations to the specified charts only. For example, to prevent access to 'Specified charts only', first authorize for 'All charts', then modify (i.e., remove) authorizations for 'Specified charts only'. Special authorizations should be used sparingly, if at all. Click on 'View only' or 'Manage' to the right of each object to authorize users to view or manage information in the indicated objects:
