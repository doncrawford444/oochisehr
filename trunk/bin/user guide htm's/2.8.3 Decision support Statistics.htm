2.8.3 Statistics

'Statistics' ('Decision Support' → 'Statistics'):
Biostatistics provides the tools to produce numerical facts and observations from health data. Specific tools are: Mean averages; perform one-way ANOVAs; t-tests; paired t-tests; signed rank tests; rank sum tests; and independent.

Program algorithms into the source code to add transcendental or arbitrarily defined functions, and for using other data (such as other databases and encoded content). Programming medical source code requires becoming familiar with the variables in the source code that can be used to formulate transcendental or arbitrarily defined functions.

Statistics tools (in combination with best practices and policy guidelines), rule making (formal heuristics that deliver medical business object requirements or specifications), and logic processes (programmed algorithms) are three levels of control to leverage health information technology. tools that add value to the robust suite of features that increase the quality of care standardizing from (NCO-certified) EHR criteria.

The 'Statistics' 'Enter data' panel displays the three dimensions of data to choose from: entry codes, time-ranges, and health records. These dimensions of data may be used with basic biostatistics to: produce distributions; test hypotheses; find averages and outliers* (when outliers are found, alerts can be triggered), and much more (see entire list below). Interesting results can be used to reduce and refine a list of potential patients whose health content may be studied.

'Encoded entries' ('Decision Support' → 'Statistics' → 'Enter data' → 'Encoded entries'):
The 'Encoded entries' panel has buttons for selecting the medical vocabulary meta-thesarus, and a field for entering a medical codes whose descriptive and encoded content are used in statistical calculations. All entered medical codes are visible in the scrollable panel below the 'Enter medical code: ' input field.

Right-click on 'Delete' to the right of any medical code (in the top panel) or group of medical codes (in the bottom panel) below to delete the medical code.

As an example, to find the mean average age for a group of patients, click, the 'HL-7 2.3.1' medical vocabulary button, enter the medical code representing a patient's birth date, then click 'Add'.

The 'Encoded entries', 'Time ranges' and 'Patient charts' input areas return a group of charts containing the specified encoded content recorded within the specified time range. This group of charts can be kept, named, and further refined.

For example, to obtain the mean average age and weight of all female patients between 18 and 65 years of age, for each year on record:

  1. Select vocabulary meta-thesaurus, 'HL-7'.
  2. Enter code 00110 for HL-7's "Date/Time of Birth".
  3. Enter code 00111 for HL-7's "Sex".
  4. Enter a "from" date/time equaling the current date - 18 years, and a "to" date/time of the current date - 65 years.

'Time ranges' ('Decision Support' → 'Statistics' → 'Enter data' → 'Time ranges'):
Time ranges specify the time of data entry (recording). A time range is from a specific date and time, to another specific date and time, such as from, "July 01, 2013 at 6:00 AM, to July 02, 2013 at 6:00 PM."

To select a from date adjust the various fields (weekday, month, day of month, and year) using arrow keys as well as keyboard input. The current date will be displayed when first entering any field. All fields adjust themselves to represent the same day, as necessary (such as, moving from "Tuesday," to, "Wednesday," also moves the calendar day up by one).

To select a from date adjust the various fields (weekday, month, day of month, and year) using arrow keys as well as keyboard input. The current date will be displayed when first entering any field.

'From date: ' ('Decision Support' → 'Statistics' → 'Enter data' → ' Time ranges ' → 'From date:'):
The 'From date' fields are labeled: 'day-of-week'; 'month'; 'day'; and 'year' (the same as the 'From date' fields). It is possible to specify the same to and from days; in this case specifying a single day.
