3.2 Security, confidentiality, and data integrity

Access to patient data is secured by password-authenticated Oochis EHR logins. All data is stored encrypted using the 256-bit AES encryption standard. All patient-identifiable data is sent encrypted in HL7 messages. Oochis EHR also provides audit tracking by recording, and never deleting, every access to data (see 'Practice Assistant' → 'Reports' → 'Logs').

Best practices for patient and provider confidentiality and data security:

  • Install firewalls on all computers hooked up to the Internet.
  • Make sure your operating system's security patches are up to date.
  • Make sure sensitive data accessed from the Internet is encrypted.
  • Use virus checking software.
  • Change passwords
Also note that threats can come from email, and insiders account for about 85% of all threats. Basic practice in this area commonly addresses:
  • The need to screen employees.
  • Do not leave confidential information such as papers, memory devices and logged-in computers (log-out when away from the computer device).
  • Destroy provider (your) data when no longer needed. Patient health data can not be deleted.
